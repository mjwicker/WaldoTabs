## Context
The `storage.session` API is used in the popup and sidebar for session-scoped state (e.g., session tokens). Firefox 109.0 didn't support it, so the lint checker was flagging the API as unsupported. Firefox 112.0 (June 2024) shipped `storage.session`, so bumping the declared minimum version to "112.0" aligns the extension's requirements with Firefox's actual capability.

## Rationale
- **Why this version?** Firefox 112 is the first stable release to ship `storage.session` natively.
- **Why add a test?** Prevents regression if the field is accidentally reverted in future cycles.
- **Why now?** Lint noise on every run is a papercut; fixing it improves developer experience and avoids install warnings at Mozilla Add-ons (AMO) submission.
