## Issues
VM-realm `Array` prototype mismatch: initial test used `assert.deepStrictEqual(resp.models, [], ...)`
which failed because arrays created inside `vm.runInContext` have a different prototype than
arrays created in the test scope. Fixed by replacing with `assert.strictEqual(resp.models.length, 0, ...)`.
This is a known Node.js vm module quirk — `deepStrictEqual` checks prototype chains.
